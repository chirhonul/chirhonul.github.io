Ideas for Bisq contributions

This page describes some ideas for potential Bisq contributions.

  • Easier setup for new developers
  • Better price feeds
  • Better fee estimation
  • Privacy improvements for light nodes

Easier setup for new developers

The documentation for new developers for Bisq is already quite good! However, starting from a minimal Linux distro, given the number of repos and their interaction, and the heavy toolchain inherent with Java, it seems that a lot of things could be simplified. Ideally, the new contributor would be able to:

  • do a clean git checkout of one repo
  • run a single command to fetch dependencies / set up environment
  • edit code in any simple editor like vim
  • run a single command to validate that all relevant code builds, and all relevant tests pass
  • push up changes
